## 1. エグゼクティブサマリー

**勤怠先生**は、飲食業界特化の勤怠異常検知＋AI是正理由文生成SaaSである。

### 市場機会
- 勤怠管理SaaS市場: 約290億円（年成長率16.7%）
- 飲食店事業所数: 約50〜60万（9割が中小企業）
- **「是正理由文の自動生成」機能を持つ競合は市場に存在しない**

### ポジショニング
```
[既存競合] 異常検知 → アラート → データ出力 → ここで終わり
[勤怠先生] 異常検知 → アラート → データ出力 → 是正理由文の自動生成 → レポート出力
                                                ↑ 市場唯一の機能
```

### 目標（コミット値 / ストレッチ目標）
| 指標 | 6ヶ月後 | 12ヶ月後 | 24ヶ月後 |
|------|---------|---------|---------|
| 登録企業数（フリー含む） | 20社 / 30社 | 80社 / 120社 | 300社 / 500社 |
| 有料顧客数 | 5社 / 10社 | 30社 / 50社 | 120社 / 200社 |
| MRR | 5万円 / 9万円 | 40万円 / 70万円 | 250万円 / 400万円 |
| チャーンレート | <5% | <3% | <2% |

> ※ MRRコミット値はフリープラン利用者が多い前提の保守値。ストレッチ値は全有料想定のシミュレーション値。

## 4. 価格戦略

### 3ティア構成

| | フリー | スタンダード | プロ |
|---|--------|------------|------|
| **月額** | **0円** | **300円/人** | **500円/人** |
| **対象** | 10名以下 | 50名以下 | 制限なし |
| **異常検知** | 全8ルール | 全8ルール | 全ルール+カスタム |
| **是正理由文生成** | 月5件まで | 無制限 | 無制限+テンプレカスタム |
| **レポート** | CSV基本出力 | CSV+PDF | CSV+PDF+API出力 |
| **検知ルール設定** | デフォルトのみ | カスタム可 | カスタム+業態別プリセット |
| **サポート** | メール | メール+チャット | 専任担当 |
| **データ保持** | 12ヶ月 | 24ヶ月 | 無制限 |

> ※ 課金対象の「従業員数」は**当月アクティブ従業員数**（1回以上の勤怠データがある従業員）で算定。飲食業界はアルバイトの入れ替わりが激しいため、登録数ではなくアクティブ数とすることで公平性を担保。

### 価格設定の根拠

- **スタンダード300円/人**: 業界標準（ジョブカン200円、KING OF TIME 300円）と同等
- **プロ500円/人**: +200円の差額は是正理由文無制限生成の価値
  - 社労士への是正対応相談: 1件あたり数万円
  - AI自動生成なら100名規模で月2万円 → 年1回の是正対応で十分にペイ
- **フリー**: PLG戦略の入口。10名以下の小規模店が対象

### 収益シミュレーション（ストレッチ目標ベース）

## 8. 法務・コンプライアンス（販売開始の前提条件）

有料SaaSを公開するために**法的に必須**の準備事項。Phase 0で最優先対応。

### 必須ドキュメント

| ドキュメント | 内容 | 優先度 |
|-------------|------|--------|
| **利用規約（ToS）** | 責任制限、SLA、データ取扱い、解約条件、免責事項 | **最優先** |
| **プライバシーポリシー** | 個人情報（従業員氏名・出退勤時刻）の取扱方針。個人情報保護法に基づく法的義務 | **最優先** |
| **特定商取引法に基づく表記** | 有料プラン提供に必要な法定表記 | **最優先** |
| **安全管理措置の説明** | データの保管・暗号化・アクセス制御の体制説明 | 高 |

### OpenAI API利用に関する法的整理

是正理由文生成時に従業員情報をOpenAI APIに送信する場合:
- **対策1**: 個人を特定できる情報（氏名等）をマスクしてからAPIに送信
- **対策2**: プライバシーポリシーに「AI処理のための第三者サービス利用」を明記
- **対策3**: OpenAIのData Processing Addendum（DPA）を確認・締結

### 決済インフラ

| 項目 | 選定 | 理由 |
|------|------|------|
| 決済基盤 | **Stripe** | SaaS課金に最適、日本語対応、従量課金対応 |
| 請求書払い | Stripe Invoice | 法人顧客向け（請求書払い需要が日本では高い） |
| 無料→有料転換 | Stripe Checkout | セルフサーブで有料プランにアップグレード |

## 12. リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| **法務未整備での公開** | **致命的** | Week 1で利用規約・プライバシーポリシーを最優先で作成 |
| **Renderコールドスタート（30秒〜1分の待機）** | **高** | デモ・初回アクセス時の信頼損失を防ぐため、有料プラン($7/月)への移行 or 定期pingで対策 |

| OpenAI APIへの個人情報送信 | 高 | 個人情報マスク処理を実装、プライバシーポリシーに明記 |
| 法令変更への対応遅れ | 中 | 社労士パートナーからの情報収集体制を構築 |
| チャーンレートが高い | 中 | オンボーディングガイド整備、初期顧客への手厚いサポート |
| CSV取り込みのみの制約 | 中 | Phase 3でAPI連携（スマレジ等）を実装し、既存システムとのダブルワーク解消 |

## 14. 成功の定義

### 3ヶ月後のマイルストーン（Go/No-Go判定）

**Go（継続・投資拡大）の条件（全て満たす）:**
- 有料顧客3社以上（または有料転換の明確なパイプライン5社以上）
- NPS 20以上
- 「是正理由文生成」機能の**月次**利用率40%以上

**Continue（継続・改善必要）の条件:**
- 有料顧客1〜2社、かつユーザーから具体的な改善要望あり
- NPS 10〜19
- 是正理由文生成の月次利用率20〜39%
- → アクション: 顧客フィードバックに基づくプロダクト改善に3ヶ月追加投資

**No-Go（ピボット検討）の条件（いずれか1つ該当）:**
- 有料転換率10%未満かつ改善の見込みなし
- NPS 10未満
- 是正理由文生成の月次利用率10%未満
